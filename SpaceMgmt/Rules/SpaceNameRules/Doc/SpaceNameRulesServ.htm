SpaceNameRulesServ

 

This Service contains an implementation of the naming rules for the Space objects in Space Management environment. This Component presently implements two naming rules for each space object namely Description rule and Space Rule. It implements the interface IJNameRule provided by GenericNameRuleSemantics component of the Common App. The IJNameRule interface is used by the generic naming rules to recompute semantic to get the naming parents and compute the naming rule name for an object. This interface is independent of the generic naming rule semantic, so the interface definition can be used even if applications are going for a new semantic.

These Naming rules are defined in the GenericNamingRules.xls sheet with its SolverProgID . When the semantic gets triggered, it does a QueryInterface for NameRuleHolder on the output of the semantic (IJNamedItem). If the flag manual is not set to true (indicating that the rule should regenerate the name), it gets the naming rule which is the ProgID for the Naming Rule Solver which is defined in xls. It then calls ComputeName on the rule(this service) which generates a new name for the SpaceChild.

 

This Component Service is comprised of eight Class modules each module-implementing name rules for space object.

AreaDescRule

This module contains an implementation of the naming rule  called description rule for the Area object in Space Management.

It implements the IJNameRule interface to recompute semantic to get the naming parents and compute the naming rule name for an object.

 

Back to Top

Methods

 

IJNameRule_ComputeName(ByVal pEntity As Object, ByVal pParents As IJElements, ByVal pActiveEntity As Object))

Description:

Generates a name for the object (area object) passed in. The name is based on the PartDescription name. It is assumed that all Naming Parents and the Object implement IJNamedItem.

  The Naming Parents are added in AddNamingParents() of the same interface. Both these methods are called from the naming rule semantic.

The generated name looks something like this: " Area description " + unique index

Parameters:

[in] pEntity

Child object that needs to have the naming rule naming.

[in] pParents

Naming parents collection.

[in] pActiveEntity

Naming rules active entity on which the NamingParentsString is stored.

 

Back to Top

 

IJNameRule_GetNamingParents(ByVal pEntity As Object) As IJElements

Description:

This method returns a reference (as pElem) to the interface of the first element in a collection of all of the Naming Parents that need to participate in the child objects naming. The parents added here are used in computing the name of the area object in the ComputeName() method of this interface. Here it is made a Dummy function which does nothing

Parameters:

[in] pEntity

Child object that needs to have the naming rule naming.
